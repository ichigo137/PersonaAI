## PersonaAI MVP — Milestone 1

**Goal:** A working chat app. Firebase email/password auth → Home screen → Chat with real AI responses (HuggingFace router, `google/gemma-3-1b-it`) → persisted to Room. Everything compiles and runs end-to-end.

Firebase project `PersonaAI` is created on the console.

---

### Step 0 — Clean slate

Delete broken/duplicate files, keep only the proven pieces:
- **Keep:** `ui/theme/` (Color, Theme, Type), `libs.versions.toml`, build files, manifest, `PersonaAIApplication.kt`.
- **Delete all:** empty placeholder files (analytics, memory, workers, datastore, use cases, data sources), `AppModule.kt`, `RepositoryProvider.kt`, `DatabaseProvider.kt`, orphaned `core/ai/providers/huggingface/` directory, duplicate `navigation/AppNavGraph.kt`, duplicate `core/network/ApiService.kt` / `AuthInspector.kt` / `NetworkMonitor.kt` / `RetrofitClient.kt`, old `features/profile`, `features/settings`.

### Step 1 — Dependencies & Firebase setup

- `gradle/libs.versions.toml` — add Firebase BoM + auth + google-services plugin versions
- Root `build.gradle.kts` — `alias(libs.plugins.google.services) apply false`
- `app/build.gradle.kts` — apply `google.services` plugin; add `firebase-bom`, `firebase-auth-ktx`
- `app/google-services.json` — *you* drop this in from Firebase console. Build compiles without it; auth won't connect until added.

### Step 2 — Core data models (Room)

`core/database/` — single `AppDatabase` with two entities:
- `Conversation` (id, title, createdAt, updatedAt)
- `Message` (id, conversationId FK, role user/assistant, content, timestamp)

`ConversationDao` + `MessageDao` with Flow queries.

### Step 3 — Network layer (HF router)

Consolidate to one clean set under `core/network/`:
- `HuggingFaceApi` → `POST v1/chat/completions`
- `HuggingFaceRequest` / `HuggingFaceResponse` DTOs
- `AuthInterceptor` (already works via BuildConfig)
- `NetworkModule` (Hilt) — OkHttp + logging + Retrofit (base `https://router.huggingface.co/`)

Model constant: `MODEL_ID = "google/gemma-3-1b-it"`.

### Step 4 — AI provider

`core/ai/`:
- `AiProvider` interface: `suspend fun chat(messages: List<ChatMessage>): String`
- `HuggingFaceProvider @Inject` — maps messages → HF request, calls API, returns text. Error handling returns message string so UI doesn't crash.

### Step 5 — Auth (Firebase)

`features/auth/`:
- `AuthRepository` interface + `FirebaseAuthRepository @Inject` (login, register, logout, currentUserFlow)
- `AuthState` — Loading / Authenticated / Unauthenticated
- `AuthViewModel` (@HiltViewModel)
- `LoginScreen` + `RegisterScreen`

### Step 6 — Chat feature

`features/chat/`:
- `domain/model/` — Conversation, Message domain models + mappers
- `domain/repository/ChatRepository` interface
- `data/ChatRepositoryImpl @Inject` — Room + AiProvider. `sendMessage(conversationId, userText)` inserts user msg, calls AI, inserts AI reply.
- `presentation/ChatViewModel` — conversation, message list, input, loading, send action.
- `presentation/ChatScreen` — ChatGPT-style UI (reuse existing ChatScreen + MessageBubble, they're already good).

### Step 7 — Home screen

`features/home/HomeScreen` — cards: Chat, History (later), Profile (later), Settings (later). Chat card creates new conversation and navigates.

### Step 8 — Navigation

`navigation/`:
- `Routes` — splash, login, register, home, chat/{conversationId}
- `PersonaNavGraph` — single NavHost. Start destination from auth state: authenticated → home, else → login.

### Step 9 — DI wiring

`di/`:
- `DatabaseModule` — AppDatabase, DAOs, repositories
- `NetworkModule` — (from step 3)
- `AiModule` — binds HuggingFaceProvider → AiProvider
- `AuthModule` — provides FirebaseAuth, binds FirebaseAuthRepository → AuthRepository

### Step 10 — MainActivity

Remove manual Room init. `@AndroidEntryPoint`, `enableEdgeToEdge()`, `setContent { PersonaAITheme { PersonaNavGraph() } }`.

---

### What you'll have
A running app: register/login (Firebase) → Home → Chat → real Gemma response from HF → messages saved to Room → history persists.

1. Add `app/google-services.json` from Firebase console (enable Email/Password sign-in).
2. Ensure `HF_API_KEY` is in `gradle.properties` (already there).
3. Build & run.
